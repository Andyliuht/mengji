# 梦迹 - 一步步部署操作指南

按顺序完成每一步，完成一步后打 ✓ 再继续下一步。

---

## 第一步：提交并推送代码到 GitHub

### 1.1 提交当前修改

在 VS Code 终端执行（PowerShell）：

```powershell
cd d:\vscode\软件工程
git add .
git status
```

确认 04-system 相关文件已被添加，然后执行：

```powershell
git commit -m "梦迹平台 - 完整功能"
```

### 1.2 推送到 GitHub

如果还没有 GitHub 仓库：

1. 打开 https://github.com/new
2. **Repository name**：`dream-journal`（或任意英文名）
3. 选择 **Public**
4. **不要**勾选 "Add a README file"
5. 点击 **Create repository**

在终端执行（将 `你的用户名` 和 `仓库名` 替换为实际值）：

```powershell
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

若已配置过 remote，直接执行：

```powershell
git push origin main
```

**若提示需要登录**：GitHub 已不支持密码，需使用 Personal Access Token  
→ GitHub → Settings → Developer settings → Personal access tokens → Generate new token

## 第二步：部署后端到 Render

### 2.1 打开 Render

1. 打开 https://dashboard.render.com
2. 用 GitHub 账号登录（Sign in with GitHub）

### 2.2 创建 Web Service

1. 点击 **New** → **Web Service**
2. 若提示连接 GitHub，点击 **Connect GitHub** 并授权
3. 在仓库列表中找到你的仓库（如 dream-journal），点击 **Connect**

### 2.3 填写配置

| 配置项 | 填写内容 |
|--------|----------|
| Name | `dream-journal-api`（可自定） |
| Region | `Singapore` 或 `Oregon` |
| **Root Directory** | `04-system/backend`（必须正确） |
| Runtime | `Node` |
| Build Command | `npm install` |
| Start Command | `npm start` |

### 2.4 添加环境变量

1. 点击 **Advanced**
2. 在 **Environment Variables** 点击 **Add Environment Variable**
3. 添加：
   - Key: `JWT_SECRET`
   - Value: `mengji-secret-2024`（可自定，建议复杂一些）

### 2.5 创建并等待

1. 点击 **Create Web Service**
2. 等待 2～5 分钟，状态变为 **Live** 即成功
3. 在页面顶部复制你的服务地址，例如：`https://dream-journal-api.onrender.com`
4. **保存这个地址**，下一步要用

### 2.6 验证后端

在浏览器打开：`https://你的地址/api/health`  
应看到 JSON：`{"status":"ok","message":"梦境记录平台 API 运行中"}`

---

- [ ] 第二步完成：后端已部署，地址已保存

## 第三步：部署前端到 Vercel

### 3.1 打开 Vercel

1. 打开 https://vercel.com
2. 用 GitHub 账号登录

### 3.2 创建项目

1. 点击 **Add New** → **Project**
2. 导入你的 GitHub 仓库（与 Render 用同一个）
3. 若未连接，点击 **Import Git Repository** 并授权

### 3.3 配置项目

| 配置项 | 填写内容 |
|--------|----------|
| Project Name | `dream-journal`（可自定） |
| **Root Directory** | 点击 Edit，填入 `04-system/frontend` |
| Framework Preset | 自动识别 Vite，无需改 |

### 3.4 添加环境变量

在 **Environment Variables** 中点击 **Add**：

- **Key**：`VITE_API_URL`
- **Value**：第二步的后端地址，如 `https://dream-journal-api.onrender.com`  
  ⚠️ **不要**加 `/api`，不要加末尾斜杠

### 3.5 部署

1. 点击 **Deploy**
2. 等待 1～2 分钟
3. 部署成功后，会显示 `https://dream-journal-xxx.vercel.app` 的地址
4. 点击该地址访问你的网站

---

- [ ] 第三步完成：前端已部署，网站可访问

## 第五步（可选）：绑定自定义域名

若希望用户通过 `mengji.com` 等域名访问：

1. 在阿里云、腾讯云等购买域名
2. 进入 Vercel 项目 → **Settings** → **Domains**
3. 添加你的域名
4. 按 Vercel 提示，在域名商的 DNS 管理中添加 CNAME 记录
5. 等待 5 分钟～48 小时生效

## 常见问题

**Q：Render 部署失败？**  
→ 检查 Root Directory 是否为 `04-system/backend`

**Q：前端能打开但登录/注册失败？**  
→ 检查 VITE_API_URL 是否正确，且没有多余 `/api` 或斜杠

**Q：首次访问很慢？**  
→ Render 免费版 15 分钟无访问会休眠，首次需约 1 分钟唤醒

**Q：数据会丢失吗？**  
→ 免费版 SQLite 在重启后可能丢失，长期使用建议升级或换云数据库
