# 梦境记录 - 数据存储说明

## 数据保存在哪里？

**所有梦境、用户、解梦结果等数据都保存在后端的 SQLite 数据库文件中。**

- **位置**：`04-system/backend/data/dreams.db`
- **类型**：SQLite 单文件数据库
- **说明**：数据持久化在本地磁盘，不会因刷新页面或重启服务而丢失

## 为什么有时数据会“消失”？

### 1. 登录状态丢失（最常见）

- 数据按**用户账号**存储，只有登录后才能看到自己的梦境
- **原因**：浏览器清除了 localStorage、使用无痕模式、或换了浏览器/设备
- **表现**：刷新后跳转到登录页，重新登录后看不到之前的记录
- **解决**：用**同一账号**登录（不要误点“注册”新建账号）

### 2. 使用了不同账号

- 若曾注册过多个账号，每个账号的数据是独立的
- **解决**：确认使用当初记录梦境时的那个用户名登录

### 3. 后端未正确连接

- 若后端未启动或端口错误，前端会请求失败
- **表现**：页面空白、一直加载、或提示“无法连接服务器”
- **解决**：确保后端在 `04-system/backend` 目录下运行 `npm run dev`

### 4. 数据库文件被删除或移动

- `dreams.db` 在 `.gitignore` 中，不会被 Git 提交
- 若手动删除 `backend/data/dreams.db`，所有数据会丢失
- **建议**：定期备份该文件（见下方）

## 如何备份数据？

直接复制数据库文件即可：

```
04-system/backend/data/dreams.db
```

建议定期复制到安全位置，例如：

```
dreams_backup_20250303.db
```

## 管理员与官方通知

- 管理员可发送**官方广播通知**给全部用户
- 设置管理员：在数据库中执行 `UPDATE users SET role = 'admin' WHERE username = '你的用户名';`
- 管理员登录后，在「消息中心」页面会看到官方通知发送表单

## 自定义数据库路径

可在 `backend` 目录的 `.env` 中设置：

```
DATABASE_PATH=D:/my-data/dreams.db
```

这样数据会保存到指定路径，便于统一备份或迁移。
