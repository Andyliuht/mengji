# 梦境记录平台 - 详细部署指南

> 从零开始，一步步将网站部署到互联网，让任何人通过网址访问

---

## 部署后效果

- 访问地址：`https://你的项目.vercel.app`（或你绑定的域名如 `mengji.cc.cd`）
- 任何人可注册、登录、记录梦境、使用全部功能

---

## 第一部分：准备工作

### 1.1 需要注册的账号（均可用 GitHub 一键登录）

| 平台 | 网址 | 用途 |
|------|------|------|
| GitHub | https://github.com | 托管代码 |
| Vercel | https://vercel.com | 托管前端 |
| Render | https://render.com | 托管后端 |

### 1.2 确认本地项目可运行

在 `d:\vscode\软件工程\04-系统实现` 目录下：

```bash
# 终端 1：启动后端
cd backend
npm install
npm run dev

# 终端 2：启动前端
cd frontend
npm install
npm run dev
```

浏览器打开 http://localhost:5173，能正常注册登录即可继续。

---

## 第二部分：将代码推送到 GitHub

### 2.1 初始化 Git 并提交

在 **VS Code 终端** 中执行（路径为 `d:\vscode\软件工程`）：

```bash
cd d:\vscode\软件工程
git init
git add .
git commit -m "梦境记录平台"
```

### 2.2 在 GitHub 创建仓库

1. 打开 https://github.com/new
2. **Repository name**：填写 `dream-journal`（或你喜欢的名字）
3. 选择 **Public**
4. **不要**勾选 "Add a README file"
5. 点击 **Create repository**

### 2.3 推送代码

在终端执行（将 `你的用户名` 替换为你的 GitHub 用户名）：

```bash
git remote add origin https://github.com/你的用户名/dream-journal.git
git branch -M main
git push -u origin main
```

若提示输入账号密码，可使用 **Personal Access Token** 代替密码（GitHub → Settings → Developer settings → Personal access tokens）。

## 第三部分：部署后端到 Render

### 3.1 创建 Web Service

1. 打开 https://dashboard.render.com 并登录
2. 点击 **New** → **Web Service**
3. 若未连接 GitHub，点击 **Connect GitHub** 并授权
4. 在仓库列表中找到 `dream-journal`，点击 **Connect**

### 3.2 配置服务

1. **Name**：`dream-journal-api`（或任意名称）
2. **Region**：选 `Singapore` 或离你最近的
3. **Root Directory**：点击 **Edit**，填入 `04-system/backend`（Render 不支持中文路径，请使用 04-system）
4. **Runtime**：`Node`
5. **Build Command**：`npm install`
6. **Start Command**：`npm start`

### 3.3 添加环境变量

1. 点击 **Advanced**
2. 在 **Environment Variables** 中点击 **Add Environment Variable**
3. 添加：
   - **Key**: `JWT_SECRET`，**Value**: `dream-journal-secret-2024`（可随意改）

### 3.4 创建并等待部署

1. 点击 **Create Web Service**
2. 等待 2～5 分钟，直到状态变为 **Live**
3. 在页面顶部找到你的服务地址，例如：`https://dream-journal-api.onrender.com`
4. **复制并保存这个地址**，后面部署前端要用

---

## 第四部分：部署前端到 Vercel

### 4.1 创建项目

1. 打开 https://vercel.com 并登录
2. 点击 **Add New** → **Project**
3. 导入你的 GitHub 仓库 `dream-journal`
4. 若未连接 GitHub，点击 **Import Git Repository** 并授权

### 4.2 配置项目

1. **Project Name**：可保持默认或改为 `dream-journal`
2. **Root Directory**：点击 **Edit**，填入 `04-system/frontend`
3. **Framework Preset**：应自动识别为 `Vite`，无需修改
4. **Environment Variables**：点击 **Add**
   - **Key**: `VITE_API_URL`
   - **Value**: 第三步的后端地址，如 `https://dream-journal-api.onrender.com`（**不要**加 `/api`）

### 4.3 部署

1. 点击 **Deploy**
2. 等待 1～2 分钟
3. 部署成功后，会显示类似 `https://dream-journal-xxx.vercel.app` 的地址
4. 点击该地址访问你的网站

---

## 第五部分：验证部署

1. 打开你的 Vercel 地址
2. 点击「立即注册」，创建账号
3. 登录后尝试记录一条梦境
4. 功能正常即表示部署成功

---

## 第六部分：绑定 DNSHE 域名（mengji.cc.cd）

### 6.1 在 Vercel 添加域名

1. 进入 Vercel 项目 → **Settings** → **Domains**
2. 在输入框填入 `mengji.cc.cd`
3. 点击 **Add**
4. 记下 Vercel 给出的解析说明（通常为 CNAME 指向 `cname.vercel-dns.com`）

### 6.2 在 DNSHE 添加解析

1. 打开 https://my.dnshe.com
2. 找到域名 `mengji.cc.cd`，点击 **+ 添加解析**
3. 添加记录：
   - **类型**：CNAME
   - **主机记录**：`@`（表示根域名）
   - **记录值**：`cname.vercel-dns.com`
4. 保存

### 6.3 等待生效

- 通常 5～30 分钟生效
- 在 Vercel 的 Domains 页面可查看状态，显示 **Valid** 即成功

### 6.4 访问

在浏览器访问：`https://mengji.cc.cd`

## 常见问题

**Q：Render 提示部署失败？**  
- 检查 Root Directory 是否为 `04-system/backend`
- 确认 Build Command 为 `npm install`，Start Command 为 `npm start`

**Q：前端能打开但注册/登录失败？**  
- 检查 Vercel 的 `VITE_API_URL` 是否正确
- 确认后端地址可访问：在浏览器打开 `https://你的后端地址/api/health`，应返回 JSON

**Q：Render 首次访问很慢？**  
- 免费版 15 分钟无访问会休眠，首次访问需约 1 分钟唤醒

**Q：域名解析一直不生效？**  
- 检查 DNSHE 记录是否正确
- 可用 https://dnschecker.org 查询域名解析状态

**Q：Render 免费版数据会丢失吗？**  
- 免费版 15 分钟无访问会休眠，重启后 SQLite 数据可能丢失
- 适合课程演示；若需长期保存数据，可考虑升级 Render 或使用 Supabase 等云数据库
