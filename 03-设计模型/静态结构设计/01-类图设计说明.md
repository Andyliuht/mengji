# 静态结构设计 - 类图设计说明

## 一、设计原则

- **面向对象**：遵循单一职责、开闭原则，模块化设计
- **分层架构**：实体层、服务层、控制层分离
- **可扩展性**：AI解梦、相似匹配等模块可独立扩展

## 二、核心类清单

### 2.1 实体类（Entity）

| 类名 | 职责 | 主要属性 |
|------|------|----------|
| User | 用户信息 | id, username, password, nickname, avatar, role, status, createdAt |
| Dream | 梦境记录 | id, userId, content, contentType, emotion, sleepQuality, isShared, shareType, createdAt |
| Tag | 标签 | id, name, category, useCount |
| DreamTag | 梦境-标签关联 | dreamId, tagId |
| DreamInterpretation | AI解梦结果 | id, dreamId, userId, content, keywords, createdAt |
| SharedDream | 分享的梦境 | id, dreamId, userId, isAnonymous, likeCount, commentCount, createdAt |
| Comment | 评论 | id, sharedDreamId, userId, content, createdAt |
| Like | 点赞 | id, sharedDreamId, userId, createdAt |
| Favorite | 收藏（分享/解梦） | id, userId, targetType, targetId, createdAt |
| Reminder | 记录提醒设置 | id, userId, remindTime, isEnabled |
| Report | 举报 | id, targetType, targetId, reporterId, reason, status, createdAt |

### 2.2 服务类（Service）

| 类名 | 职责 |
|------|------|
| DreamService | 梦境的增删改查、分享、导出 |
| DreamInterpretationService | AI解梦、解梦结果管理 |
| SimilarDreamService | 相似梦境匹配算法 |
| TagService | 标签管理、推荐标签 |
| StatisticsService | 个人梦境统计、可视化数据 |
| ExportService | 梦境导出（PDF/文本） |
| ReminderService | 提醒设置、定时任务 |

### 2.3 控制类（Controller）

| 类名 | 职责 |
|------|------|
| DreamController | 梦境相关 API |
| CommunityController | 分享、评论、点赞、收藏 API |
| InterpretationController | AI解梦、相似匹配 API |
| UserController | 用户、提醒设置 API |
| AdminController | 内容审核、用户管理 API |

## 三、类之间的关系

- **User 1 --- * Dream**：用户拥有多条梦境
- **Dream * --- * Tag**（通过 DreamTag）：梦境与标签多对多
- **Dream 1 --- 0..1 DreamInterpretation**：梦境可有解梦结果
- **Dream 1 --- 0..1 SharedDream**：梦境可被分享
- **SharedDream 1 --- * Comment**：分享梦境有多条评论
- **SharedDream * --- * User**（通过 Like）：点赞多对多
- **User 1 --- * Favorite**：用户可收藏多条内容
- **User 1 --- 1 Reminder**：用户有一条提醒设置
